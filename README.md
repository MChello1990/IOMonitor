# IO Monitor — 硬盘 I/O 使用状况监视工具

## 概述

IO Monitor 是一款 Windows 平台下的硬盘 I/O 监视工具。它能够实时监控每个进程的磁盘读写情况，生成速率排行与会话累计统计，并以彩色控制台界面（CUI）展示。

**核心特点：**
- 实时监测各进程磁盘读/写速率
- 多维度排行（读、写、总速率、会话累计、进程生命周期）
- 物理磁盘吞吐统计
- **CSV 录制** — 支持将 I/O 数据实时导出为 CSV 文件
- **悬浮窗模式** — 25% 透明度的置顶迷你窗口，可快速查看当前磁盘活动
- 极低的 CPU 和内存占用，适合长期后台运行
- 控制台原生彩色界面，无需图形环境

## 快速开始

### 编译

**MSVC（Visual Studio）：**

直接打开 `IOMonitor.slnx` 编译，或使用 CMake：

```bash
cmake -B build -G "Visual Studio 17 2022"
cmake --build build --config Release
```

**MinGW-w64：**

```bash
g++ -std=c++17 -O2 -DUNICODE -D_UNICODE \
    -DWIN32_LEAN_AND_MEAN -DNOMINMAX \
    -o iomonitor.exe src/main.cpp src/Monitor.cpp src/Display.cpp \
    src/Recorder.cpp src/OverlayWindow.cpp \
    -lpdh -lpsapi -lgdi32 -municode -mconsole -static
```

### 运行

```bash
# 默认参数运行（采样 1000ms，刷新 500ms，显示前 30 进程）
iomonitor.exe

# 自定义参数
iomonitor.exe -s 500 -r 300 -n 50

# 启动时自动开始 CSV 录制
iomonitor.exe -o

# 查看帮助
iomonitor.exe --help
```

> **提示：** 以管理员权限运行可获得完整的进程覆盖范围，否则部分系统进程的 I/O 数据无法读取。

## 排序模式详解

| 模式 | 触发键 | 说明 | 适用场景 |
|------|--------|------|----------|
| **Total Rate** | `T` | 按当前读+写速率排序 | 查看实时磁盘活动最活跃的进程 |
| **Read Rate** | `R` | 按当前读取速率排序 | 排查大量读取的进程 |
| **Write Rate** | `W` | 按当前写入速率排序 | 排查大量写入的进程 |
| **Session IO** | `S` | 按监视期间的累计 I/O 排序 | 查看长期磁盘使用量最大的进程 |
| **Process IO** | `P` | 按进程自启动以来的总 I/O 排序 | 查看历史上磁盘 I/O 最多的进程 |

## CSV 录制功能

按 `O` 键或使用 `-o` 启动参数可开启 CSV 录制。录制文件保存在可执行文件同目录下的 `records/` 文件夹中，文件名格式为 `IO_YYYYMMDD_HHMMSS.csv`。

### CSV 列定义

| 列名 | 说明 |
|------|------|
| `Timestamp` | 采样时间戳（ISO 8601 格式） |
| `PID` | 进程 ID |
| `ProcessName` | 进程可执行文件名 |
| `ProcessPath` | 进程完整镜像路径 |
| `ReadRate(B/s)` | 当前读取速率（字节/秒） |
| `WriteRate(B/s)` | 当前写入速率（字节/秒） |
| `TotalRate(B/s)` | 总速率（读+写） |
| `SessionReadBytes` | 会话累计读取字节数 |
| `SessionWriteBytes` | 会话累计写入字节数 |
| `SessionTotalBytes` | 会话累计总 I/O 字节数 |
| `ProcessReadBytes` | 进程生命周期读取字节数 |
| `ProcessWriteBytes` | 进程生命周期写入字节数 |
| `Event` | 事件类型（`sample` / `start` / `stop`） |

### 录制事件类型

- `start` — 录制开始
- `sample` — 周期性采样快照（每个采样间隔写入一次）
- `stop` — 录制停止

## 悬浮窗模式

按 `M` 键进入悬浮窗模式。悬浮窗是一个 25% 透明度的置顶迷你窗口，显示：

- 当前总磁盘 I/O 速率（读 + 写）
- 磁盘使用百分比
- 录制状态指示
- 活跃进程数

**操作：**
- **鼠标悬停** — 悬浮窗变为 80% 不透明度
- **鼠标移出** — 恢复 25% 透明度
- **双击悬浮窗** — 返回完整控制台模式
- **再次按 `M`** — 返回完整控制台模式

## 技术原理

### 数据采集

1. **进程枚举** — 通过 `CreateToolhelp32Snapshot` 获取当前所有运行进程的快照
2. **I/O 计数器** — 对每个进程调用 `GetProcessIoCounters` 获取累积 I/O 计数（`IO_COUNTERS`）
3. **速率计算** — 对比前后两次采样的计数值，除以时间间隔得到速率
4. **物理磁盘** — 通过 PDH（Performance Data Helper）查询 `\PhysicalDisk(_Total)\Disk Read/Write Bytes/sec` 性能计数器
5. **会话累计** — 维护每个进程自监视启动以来的 I/O 增量总和
6. **CSV 录制** — 异步写入队列，独立线程将采样数据写入 CSV 文件，不阻塞主循环
7. **悬浮窗** — 独立 Win32 窗口线程，支持 WS_EX_LAYERED 分层窗口实现透明度控制

### 性能设计

- 监视线程以 `THREAD_PRIORITY_LOWEST` + `THREAD_MODE_BACKGROUND_BEGIN` 运行，不会与前台应用争抢 CPU
- 每次采样仅需 ~3 次 API 调用/进程（`OpenProcess` → `GetProcessIoCounters` → `CloseHandle`），200 进程约 <10ms
- 可执行文件约 ~214KB，内存占用约 2-3MB
- CSV 录制使用异步写入队列（最大 200 条缓存），独立线程写入，零阻塞
- 支持 PID 重用检测，进程退出后 PID 被复用时自动重置计数
- 控制台界面使用差分更新（row cache），仅重绘变化行

### 兼容性

- 操作系统：Windows 10 版本 1607 及以上（需 VT100 终端支持）
- 编译器：MSVC 2019+ 或 MinGW-w64 8.0+
- C++ 标准：C++17

## 常见问题

### Q: 为什么某些进程没有显示？

部分系统进程（如受保护的反恶意软件进程）可能无法通过 `PROCESS_QUERY_LIMITED_INFORMATION` 打开。以**管理员权限**运行可以获得更完整的进程覆盖。

### Q: 为什么物理磁盘速率显示为 0？

如果 PDH 计数器初始化失败（例如在虚拟机或某些精简版 Windows 上），物理磁盘速率将显示为 0。此时程序仍可正常显示各进程的 I/O 数据。

### Q: 会话累计与进程生命周期 I/O 有何区别？

- **会话累计**：仅统计监视器运行期间的 I/O 量，按 `C` 可清零重新计数
- **进程生命周期 I/O**：进程自启动以来的全部 I/O 量，反映了进程的历史磁盘使用

### Q: 如何降低采样开销？

增大采样间隔（如按 `5` 切换到 5000ms 模式）可进一步降低 CPU 占用。对于长期后台监视场景，推荐 2000–5000ms 采样间隔。

### Q: CSV 录制文件保存在哪里？

录制文件保存在可执行文件所在目录下的 `records/` 子文件夹中。程序启动时会自动创建该目录。

### Q: 悬浮窗模式下如何返回完整界面？

双击悬浮窗或再次按 `M` 键即可返回完整控制台模式。

### Q: 支持远程监视吗？

当前版本仅支持本地监视。可通过管道或网络扩展实现远程监视（WIP）。
